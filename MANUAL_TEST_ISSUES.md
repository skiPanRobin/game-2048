## 问题 1：移动端页面可滚动导致游戏操作冲突

**问题描述**：
使用手机浏览器打开后，整个游戏页面可以被手指左右/上下滑动。这是因为浏览器默认启用了橡皮筋滚动效果（overscroll behavior），当手指在屏幕上滑动时，页面会随之滚动。这与游戏的移动操作（方向键/滑动手势）产生冲突，导致移动指令不清晰。

**解决步骤**：

1. 在 `src/index.css` 或全局样式中添加以下 CSS：
   ```css
   html, body {
     overscroll-behavior: none;
     overflow: hidden;
     touch-action: none;
   }
   ```

2. 或在 `src/components/GameBoard.tsx` 中为游戏容器添加：
   ```css
   .game-container {
     overscroll-behavior: none;
     touch-action: none;
   }
   ```

3. 或者在 `index.html` 的 `<body>` 标签添加：
   ```html
   <body ontouchstart="">
   ```

**推荐方案**：使用 CSS 的 `overscroll-behavior: none` 和 `touch-action: none`，这是最干净的无侵入式解决方案。

**优先级**：高 - 影响移动端核心体验

**状态**：[x] 已完成

## 问题 3：移动端上下滑动方向反向

**问题描述**：
在移动端游戏中，左右滑动正确，但上下滑动是反直觉的。用户向上滑动时，棋子向下移动；向下滑动时，棋子向上移动。

**问题分析**：
这个问题与问题4（PC端方向反向）实际上是相互关联的，因为移动端和键盘都使用同一个 `merge` 函数来处理移动逻辑。必须确保两者的方向映射一致，才能同时正常工作。

**解决步骤**：

1. 检查 `src/hooks/useSwipe.ts` 中的方向映射逻辑
2. 在 `handleTouchEnd` 函数中使用正向映射：
   ```typescript
   // Vertical swipe
   // Direct mapping: swipe up (negative deltaY) → 'up', swipe down (positive deltaY) → 'down'
   if (deltaY > 0) {
     onSwipe('down');
   } else {
     onSwipe('up');
   }
   ```

**关键点**：
- deltaY > 0 表示手指向下滑动（屏幕内容向上移动）
- deltaY < 0 表示手指向上滑动（屏幕内容向下滑动）
- 使用直接映射：滑动方向 = 棋子移动方向

**状态**：[x] 已完成

## 问题 4：PC端上下箭头方向反向

**问题描述**：
在 PC 端使用键盘方向键时，上下箭头的方向是反直觉的。按下向上箭头时，棋子向下移动；按下向下箭头时，棋子向上移动。

**问题分析**：
这个问题与问题3（移动端滑动反向）相互关联。由于两者共用同一个 `merge` 函数，需要确保键盘输入和触摸输入的方向映射一致。

**解决步骤**：

1. 检查 `src/game/logic.ts` 中的 `getRotationCount` 函数
2. 设置正确的旋转次数：
   - 'up': 3 (270° 旋转，使格子向上合并到顶部 row 0)
   - 'down': 1 (90° 旋转，使格子向下合并到底部 row 3)

**关键点**：
- 旋转 0 次：向左合并
- 旋转 1 次 (90°)：向下合并
- 旋转 2 次 (180°)：向右合并
- 旋转 3 次 (270°)：向上合并
- 'up' 方向需要旋转 3 次让格子"左边"对齐，然后合并到左边即顶部
- 'down' 方向需要旋转 1 次让格子"左边"对齐，然后合并到左边即底部

**状态**：[x] 已完成

---

## 问题 3 & 4 总结：方向映射的一致性

**核心问题**：
移动端滑动和 PC 键盘共用同一个游戏核心逻辑（`merge` 函数），必须保持两者的方向映射一致。

**最终解决方案**：
1. `src/game/logic.ts` - getRotationCount：up=3, down=1
2. `src/hooks/useSwipe.ts` - 方向映射：deltaY>0→'down', deltaY<0→'up'

两处都使用"符合直觉"的映射方式，确保：
- 按下/滑向"上" → 棋子向移动
- 按下/滑向"下" → 棋子向下移动
